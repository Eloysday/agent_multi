# 二、ROS、图文数据流传输 核心内容，哪些必须学、哪些浅了解

## 1）ROS2 必学（应用集成岗刚需，类比分布式消息中间件）

ROS2 本质是**机器人本地分布式微服务通信框架**，和你 Kafka、微服务思维完全互通：

1. 核心概念（对应你已掌握的技术）

    - Node 节点 = 独立微服务进程

    - Topic 话题 = 单向消息队列（类似 Kafka Topic）

    - Service 服务 = 同步 HTTP 接口调用

    - Action = 长时异步任务（类似 Celery）

    - DDS 通信 = 本地低延迟消息分发，QoS 对应消息可靠性策略（acks / 死信）

2. 必须落地的工程能力（写进项目）

    - rclpy Python 节点开发，发布 / 订阅图像、传感器数据流；

    - rosbridge WebSocket 桥接 ROS 与云端 FastAPI，打通本地机器人与远程服务；

    - 多机 ROS 分布式通信、时间戳对齐，解决图文数据时序错乱；

    - 二进制图像流压缩、分片传输，降低带宽占用。

3. 浅了解不用深挖：
MoveIt 运动规划、URDF 建模、底层驱动、实时内核优化（控制岗内容，集成岗无需精通）。

## 2）多模态图文数据流传输（你项目核心加分模块，完全贴合你的两台轻量服务器）

### 完整链路（可直接做成简历核心项目）

机器人端 ROS2 → 图像 / 点云 / IMU 二进制流 → Kafka 实时流队列
→ 分流两套服务：

1. **实时监控流**：机器 A FastAPI 消费，缩略图存入 MinIO，元数据存入 MongoDB（Beanie），前端实时看板展示；

2. **AI 推理流**：转发至机器 B 独立算力节点，多进程 OpenCV\+YOLO 检测 \+ Embedding 向量化，结果回写 Mongo、存入向量库 RAG；

3. **LLM 具身 Agent**：用户自然语言指令 → Agent 检索机器人图文历史数据 → LLM 拆解动作指令 → 反向下发 ROS2 执行任务。

1. 二进制大文件流式分片传输、内存缓冲，防止 OOM；

2. 统一时间戳对齐多传感器图文数据，解决时序错位；

3. 流量削峰：Kafka 缓冲机器人高频上报，避免瞬时打垮数据库；

4. 冷热分层存储：实时图像存 MinIO，结构化检测结果存 Mongo，短期缓存 Redis；

5. 多进程并行处理图像推理、批量向量化，充分利用多核 CPU。
